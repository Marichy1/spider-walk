# 蜘蜘（Zhizhu）—— 维护与代码阅读指南

## 1. 项目概览
“蜘蜘”是一个单页复古漫游网站，核心交互是：  
- 中心蜘蛛形象会注视鼠标/标签，点击后生成随机标签。  
- 标签在页面上移动，悬停即停，可拖拽，点击弹出推荐窗口。  
- 弹窗可拖动，展示随机推荐的外部网站信息（支持自动获取网站图标）。  
- 移动端适配：蜘蛛视线自动跟随标签，弹窗蜘蛛注视下方，触摸交互优化。

整个项目由单个 HTML 文件构成，包含 HTML 结构、CSS 样式、JavaScript 逻辑三部分。

## 3. 数据配置
### 3.1 页面数据 (PAGES)
数组 `PAGES` 存储所有可推荐的网站。每个对象字段：
- `name` : 网站名称
- `desc` : 简短介绍
- `img`   : 图片设置  
  - `"auto"` — 自动获取网站 `/favicon.ico`  
  - `"placeholder"` — 显示带标签文字的灰色方块  
  - 自定义图片 URL  
- `url`   : 网站链接
- `tags`  : 所属标签数组，如 `["#音乐", "#设计"]`

### 3.2 标签池 (TAG_POOL)
数组 `TAG_POOL` 定义了首页可能出现的所有标签类型。初始为空数组，通过编辑器维护。蜘蛛点击生成的随机标签也从该池中抽取。

### 3.3 修改数据的方式
使用配套编辑器 `editor.html` 可视化修改，或直接修改代码中的 `PAGES` / `TAG_POOL` 数组。

## 4. 蜘蛛绘制系统
### 4.1 核心参数
`SPIDER_BODY` 对象定义了主页蜘蛛的身体坐标、半径和腿的几何参数：
```js
const SPIDER_BODY = { x:120, y:120, r:30, seg1:10, seg2:20, bend:1.5 };
```
- `x, y` — 圆心在 `spider-canvas` 画布上的坐标  
- `r` — 身体半径  
- `seg1, seg2` — 腿第一段、第二段长度  
- `bend` — 腿的弯折弧度

### 4.2 眼睛布局
眼睛为水平排列的四圆（小-大-大-小），紧贴相切，圆心 Y 坐标与身体圆心一致。  
`EYE_RADII` 定义半径，`EYE_CX` 定义相切后的圆心 X 偏移。

### 4.3 瞳孔跟随
瞳孔位置根据目标点（鼠标或虚拟视线）和眼睛圆心的相对方向计算，移动范围限制在半径 40% 内。

### 4.4 弹窗蜘蛛
弹窗上的蜘蛛为缩小版，参数由 `POPUP_SPIDER` 定义，缩放比例根据 `br/30` 计算。移动端时瞳孔始终注视正下方。

### 4.5 绘制函数
- `drawSpider()` — 绘制完整蜘蛛（身体、腿、眼睛）
- `drawEye()` — 绘制单个眼睛
- `drawMainSpider()` — 主页蜘蛛的封装（视线冻结、移动端虚拟视线）
- `drawPopupSpider()` — 弹窗蜘蛛封装

## 5. 交互系统
### 5.1 标签交互
- **生成**：点击蜘蛛身体、弹窗关闭、通过标签拖拽松手或直接调用 `addTag()`。
- **移动**：每个标签具有随机速度（30~60 px/s），在 `layer-container` 内反弹。
- **悬停暂停**：桌面端由 `isMouseOverTag()` 每帧检测并设置 `dataset.hovered`。
- **拖拽**：使用 Pointer Events 统一处理鼠标与触摸。按下后跟踪移动，超过阈值（移动端 10px，桌面 3px）判定为拖拽。松手后若是拖拽则重新随机速度，否则触发点击推荐。
- **蜘蛛身体排斥**：标签按下时若触摸点在蜘蛛身体圆内则直接不响应，由蜘蛛处理。

### 5.2 蜘蛛点击
- 全局 `pointerup` 事件中检测是否在 `SPIDER_BODY.r` 范围内。
- 仅当没有标签拖拽、弹窗未显示且指针在身体圆内时触发。
- 触发时瞳孔播放放大-缩小动画，然后生成一个随机标签到点击位置。

### 5.3 弹窗交互
- **打开**：点击标签或“随机漫游”按钮 → `recommendRandom()` → 显示遮罩 + 弹窗（带缩放动画）。
- **图片加载**：立即显示“寻找图片中…”，异步加载 favicon.ico。成功则替换，失败或超时则显示“图片被吃掉啦！”。
- **拖动**：顶栏 `pointerdown` 捕获，移动时更新弹窗位置，限制在视口内。
- **关闭**：点击 × 按钮或点击遮罩 → `closePopup()`。关闭前会将被点击的标签类型生成一个新标签到主页。

### 5.4 “随机漫游”按钮
点击后从 `TAG_POOL` 中随机选一个标签调用推荐。

## 6. 移动端适配
### 6.1 设备检测
`isMobile` 变量基于 `'ontouchstart' in window` 或 `navigator.maxTouchPoints` 判断。

### 6.2 蜘蛛视线
- 移动端不跟随鼠标，改为自动选择目标：  
  - 有标签拖拽时注视被拖拽标签  
  - 否则随机选择一个标签平滑追随  
  - 无标签时注视“随机漫游”按钮  
- 切换间隔 3~5 秒，视线平滑插值移动。

### 6.3 弹窗蜘蛛
移动端弹窗上的蜘蛛瞳孔固定指向正下方（弹窗内容区）。

### 6.4 触摸交互
- 使用 Pointer Events 统一处理，拖拽阈值提高到 10px 防止误触。
- 标签按压时阻止默认行为和冒泡，确保不会触发蜘蛛事件。

### 6.5 样式适配
`<meta name="viewport">` 设置 `user-scalable=no`，`body` 加 `touch-action: manipulation`，防止双击缩放。

## 7. 动画与视觉效果
- **标签生成动画**：`@keyframes tag-pop-in` 缩放淡入效果。
- **弹窗弹出动画**：`transform: scale(0)→scale(1)` + `opacity` 过渡。
- **遮罩淡入淡出**：`opacity` 过渡。
- **蜘蛛瞳孔动画**：点击后 `requestAnimationFrame` 循环放大再缩小。
- **背景网格**：Canvas 绘制亮灰色小点。

## 9. 修改与扩展指南
- **更改配色**：修改 `:root` 中的 CSS 变量。
- **添加新标签/网站**：推荐使用 `editor.html`；也可直接修改 `PAGES` 和 `TAG_POOL`。
- **调整蜘蛛大小**：修改 `SPIDER_BODY.r`，眼睛和腿会自动等比缩放。
- **修改弹窗图片逻辑**：编辑 `loadPopupImage()` 中的 `resolveFaviconUrl()` 可换成其他图标源。
- **改变标签速度范围**：修改 `randomSpeed()` 中的 `30 + Math.random() * 30`。
- **移动端视线行为**：调整 `updateVirtualGaze()` 中的插值系数或切换间隔。

## 10. 常见问题
1. **标签点击无反应** → 检查标签的 `z-index` 是否低于蜘蛛画布，或拖拽阈值过大导致移动端被误判为拖拽。
2. **弹窗一闪而过** → 通常由于 `click` 事件延迟触发导致关闭，本版已改用 `pointerup` 解决。
3. **图片一直显示“寻找图片中…”** → 目标网站可能无 `favicon.ico` 或跨域限制，已内置超时回退。
4. **移动端拖拽后标签停止** → 确保 `pointerup` 中移动端强制清除 `hovered` 状态。
