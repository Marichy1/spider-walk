\# 蜘蜘（Zhizhi）



\*\*带你随便逛逛。\*\*



\---



\## 项目简介



“蜘蜘”是一个随机网络导航网站，收录了大量\*\*风格独特、不循常规\*\*的网页。它们不追求实用，却有着某种让人停留的气质——可能是古怪的交互、过时的界面、晦涩的谜题，或只是某种难以言说的氛围。



中心的跳蛛“蜘蜘”是向导，也是守护者。点击它，或者拖动周围的标签，它就会带你去一个你从未去过的地方。



\---

\## 视觉风格



整体风格致敬\*\*古早互联网时期\*\*的个人主页。



\- \*\*字体\*\*：使用等宽打字机字体栈（Courier New, Liberation Mono 等），跨平台均可正常显示。

\- \*\*配色\*\*：以纯白、纯黑、灰白（#F2F2F2）为主，辅以不同亮度的灰色用于边框与装饰，视觉上干净、克制。

\- \*\*元素\*\*：白色矩形标签、点阵背景、黑色线条框、硬阴影——还原早期网页的“手工感”与“模拟感”。

\- \*\*动画\*\*：标签生成时有轻微弹跳，弹窗以缩放动画出现，蜘蛛瞳孔有独立的放大-收缩反馈。所有动效克制、不喧闹。

\## 交互设计



网站围绕“探索”的隐喻展开：



\- \*\*漫游\*\*：点击“随机漫游”按钮，蜘蜘随机推荐一个页面。

\- \*\*标签\*\*：可拖拽、可点击。点击某一标签，蜘蜘会尽量找相关页面。关闭弹窗后，页面所带的所有标签会飘到主页上。

\- \*\*蜘蜘本体\*\*：位于画面中央。眼睛跟随鼠标（桌面端）或随机注视标签（移动端）。点击身体会生成一个新标签并播放瞳孔动画。

\- \*\*弹窗\*\*：展示推荐站点的名称与简介。顶栏可拖拽，图片自动尝试获取网站图标（失败则显示“图片跑掉啦！”）。

\- \*\*移动端适配\*\*：提升了拖拽阈值以防止误触，视线自动追随标签，弹窗尺寸适应小屏幕。

\## 选材标准



收录的网站大致符合以下某种特质：



\- \*\*技术怀旧\*\*：早期网页风格、Flash 时代残留、DOS 界面模拟

\- \*\*实验艺术\*\*：交互装置、生成艺术、数字拼贴

\- \*\*谜题与 ARG\*\*：网页解谜、替代现实游戏、文字推理

\- \*\*氛围感\*\*：不安、静谧、荒诞、忧伤等情绪营造

\- \*\*冷门社区\*\*：已没落或长期小众的论坛、博客、BBS

\- \*\*无意义但有趣\*\*：纯粹为了好玩而存在的网页



> 标准不是“好不好用”，而是“是否值得被记住”。

\## 数据维护



网站中展示的所有推荐内容均存储在 `index.html` 文件内的 `PAGES` 数组中：



```js

const PAGES = \[

&#x20; {

&#x20;   "name": "示例网站",

&#x20;   "desc": "一段简短介绍。",

&#x20;   "img": "auto",

&#x20;   "url": "https://example.com/",

&#x20;   "tags": \["#艺术", "#神秘"]

&#x20; },

&#x20; ...

];

```



\- `img` 设为 `"auto"` 时自动获取网站 favicon.ico；也可换成 `"placeholder"` 或自定义图片链接。

\- `tags` 对应标签池 `TAG\_POOL` 中的标签名（以 `#` 开头）。



更便捷的维护方式：用浏览器打开 `editor.html`，上传 `index.html` 即可可视化增删改标签与页面，然后下载更新后的文件。

\## 理念



互联网上有很多“没什么用”的角落。它们不是为了盈利而存在，不追求流量，不关心 SEO。它们只是某些人某个时刻的某种表达。



“蜘蜘”想做的，是在这些角落之间编织一条若隐若现的丝线，让偶然踏足的人，能够遇见。



名字来源于跳蛛（jumping spider）——一种好奇心旺盛的小生物。它们不像其它蜘蛛那样结网等待，而是\*\*主动跳出去探索\*\*。
